# LED Video Screening Tool

This repository provides a supplementary video-based screening tool for urban LED display screens. It implements the practical video-screening layer described in the manuscript and is intended for preliminary identification of high-priority display screens for review.

The tool does not calculate calibrated illuminance, luminance, melanopic equivalent daylight illuminance, melanopic daylight efficacy ratio, BLER, or health risk. Its outputs are video-derived screening proxies and management-oriented flags.

## What The Tool Does

- Annotates target-screen, other-screen, and excluded polygons on sampled video frames.
- Extracts video-derived indicators from the annotated target screen and nearby background.
- Calculates luminance-proxy, blue-channel, saturation, chromaticity-occupancy, and temporal-variation indicators.
- Applies reference screening thresholds derived from an 85-video Shanghai commercial-district dataset.
- Produces CSV outputs and a one-page dashboard image for each processed video.

## Main Outputs

- `video_indicator_summary.csv`: video-level extracted indicators.
- `screening_results.csv`: screening flags, scores, profile suggestions, and management suggestions.
- `screening_thresholds.csv`: thresholds used for each screening dimension.
- `roi_quality_summary.csv`: basic ROI quality information.
- `dashboards/`: image dashboards for quick review.
- `quality_overlays/`: ROI overlay images for checking polygon quality.

## Screening Dimensions

The tool screens five video-derived dimensions:

- High luminance-proxy content.
- Temporal variability.
- Blue-channel tendency.
- Screen-background contrast.
- Chromatic spread.

These are screening dimensions, not regulatory standards or confirmed health-risk thresholds.

## Installation

Use Python 3.10 or later.

```bash
pip install -r requirements.txt
```

## Quick Use In PyCharm

For a simple local workflow, edit the settings near the top of `led_video_screening.py`:

```python
SIMPLE_VIDEO_PATH = r".\example_videos"
SIMPLE_MODE = "annotate_extract_screen"
```

`SIMPLE_VIDEO_PATH` can point to either a folder containing videos or a single video file. The default extraction interval is 0.5 s, matching the manuscript's dynamic-analysis sampling scale more closely than a 1 s interval.

Run:

```bash
python led_video_screening.py
```

The script creates annotation and result folders next to the input video folder.

## Command-Line Workflow

```bash
python led_video_screening.py annotate --video-dir "path/to/videos" --annotation-dir "path/to/annotations"
python led_video_screening.py extract --video-dir "path/to/videos" --annotation-dir "path/to/annotations" --out-dir "path/to/results"
python led_video_screening.py screen --indicator-csv "path/to/results/video_indicator_summary.csv" --out-dir "path/to/results" --use-reference-thresholds
```

## Annotation Controls

- Left click: add polygon point.
- `1`: target screen.
- `2`: other screen to exclude from background.
- `3`: excluded area.
- Enter or Space: confirm polygon.
- `Z`: undo point.
- `X`: undo polygon.
- `N`: save current frame and move to next sampled frame.
- `Q`: save current video and move to next video.
- Esc: stop.

## Reference Thresholds

The default `--use-reference-thresholds` option uses upper-quartile or rule-based thresholds from the 85-video study dataset. These thresholds are intended for relative screening and priority identification. For a new city, camera type, or management context, users should recalibrate thresholds using a local reference sample.
